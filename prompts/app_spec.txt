<project_specification>
  <project_name>魔塔 (Magic Tower) - Classic RPG Puzzle Game</project_name>

  <overview>
    Build a fully functional classic Magic Tower (魔塔) web game. Magic Tower is a traditional Chinese
    turn-based RPG puzzle game where the player navigates through a multi-floor tower, collecting items,
    fighting monsters, and solving strategic route puzzles to reach the top. The game features grid-based
    movement, turn-based combat (bump-to-attack), key-and-door mechanics, and RPG stat progression.
    The UI should feature a retro pixel-art style with a clean, modern web implementation.
  </overview>

  <technology_stack>
    <frontend>
      <framework>React with Vite</framework>
      <styling>Tailwind CSS (via CDN) + custom CSS for pixel-art sprites</styling>
      <state_management>React hooks, context, and useReducer for game state</state_management>
      <rendering>HTML5 Canvas or CSS Grid for tile map rendering</rendering>
      <port>Only launch on port {frontend_port}</port>
    </frontend>
    <backend>
      <runtime>Node.js with Express</runtime>
      <database>SQLite with better-sqlite3 (for save/load game state)</database>
    </backend>
    <architecture>
      Pure frontend game logic with optional backend for persistence.
      All game mechanics run client-side for instant responsiveness.
      Backend handles save/load, leaderboard, and user preferences.
    </architecture>
  </technology_stack>

      - Install backend dependencies as needed

  <game_design>
    <game_world>
      The tower consists of multiple floors (typically 50 floors in the classic game, but we implement
      a complete playable version with at least 11 floors as the initial content). Each floor is a
      grid-based map (typically 11x11 tiles per floor). The player starts at the bottom floor and
      must ascend to the top.

      <floor_types>
        - Normal floors: contain monsters, items, keys, and doors
        - Shop floors: NPCs that buy/sell items
        - Boss floors: contain powerful boss monsters guarding progress
        - Special event floors: unique puzzles or story events
      </floor_types>

      <tile_types>
        - Wall (impassable)
        - Floor (passable)
        - Stairs up (to next floor)
        - Stairs down (to previous floor)
        - Yellow door (requires yellow key)
        - Blue door (requires blue key)
        - Red door (requires red key)
        - Yellow key (pickup item)
        - Blue key (pickup item)
        - Red key (pickup item)
        - Small potion (HP +200)
        - Big potion (HP +500)
        - Super potion (HP +1000 or full heal)
        - Red gem (ATK +3)
        - Blue gem (DEF +3)
        - Green gem (ATK +1, DEF +1)
        - Sword (ATK +10 or other fixed amount)
        - Shield (DEF +10 or other fixed amount)
        - Attack book (ATK + permanent upgrade)
        - Defense book (DEF + permanent upgrade)
        - Life gem (max HP +)
        - Gold pile (gold +50)
        - Big gold pile (gold +200)
        - Monster (various types, see monster_types)
        - NPC (interactive characters)
        - Shopkeeper (buy/sell items)
        - Hidden wall (looks like wall but is passable - revealed on adjacent)
      </tile_types>
    </game_world>

    <player_stats>
      - HP (Hit Points): current health, reach 0 = game over
      - Max HP: maximum health capacity
      - ATK (Attack): damage dealt per hit
      - DEF (Defense): damage reduced per hit
      - Gold: currency for shops
      - Yellow Keys: count of yellow keys held
      - Blue Keys: count of blue keys held
      - Red Keys: count of red keys held
      - Floor: current floor number
      - Steps: total steps taken (used for scoring)
    </player_stats>

    <combat_system>
      Combat is turn-based and automatic (bump-to-attack):
      - Player moves into a monster tile to initiate combat
      - Combat is auto-resolved: player attacks first, then monster attacks if alive
      - Damage to monster = Player ATK - Monster DEF (minimum 0)
      - Damage to player = Monster ATK - Player DEF (minimum 0)
      - Combat continues until monster HP or player HP reaches 0
      - If player wins: monster removed from map, player keeps remaining HP
      - If player loses: game over screen with option to restart or load save
      - Display damage numbers and combat result

      <example>
        Player (HP:1000, ATK:20, DEF:10) vs Green Slime (HP:50, ATK:15, DEF:5)
        Round 1: Player deals 20-5=15 damage (Slime HP: 35), Slime deals 15-10=5 damage (Player HP: 995)
        Round 2: Player deals 15 damage (Slime HP: 20), Slime deals 5 damage (Player HP: 990)
        Round 3: Player deals 15 damage (Slime HP: 5), Slime deals 5 damage (Player HP: 985)
        Round 4: Player deals 15 damage (Slime HP: 0), Slime defeated!
        Player wins with HP: 985, took 4 rounds
      </example>
    </combat_system>

    <monster_types>
      Classic monster hierarchy (stats should be balanced for strategic gameplay):

      Floor 0-3 (Easy):
      - Green Slime: HP 35, ATK 18, DEF 1
      - Red Bat: HP 45, ATK 20, DEF 5
      - Skeleton: HP 50, ATK 22, DEF 8
      - Magician: HP 40, ATK 25, DEF 3

      Floor 4-6 (Medium):
      - Red Slime: HP 70, ATK 30, DEF 10
      - Stone Golem: HP 100, ATK 35, DEF 20
      - Dark Knight: HP 120, ATK 40, DEF 25
      - Witch: HP 80, ATK 45, DEF 15

      Floor 7-9 (Hard):
      - Vampire: HP 200, ATK 55, DEF 30
      - Dragon: HP 300, ATK 60, DEF 40
      - Dark Mage: HP 250, ATK 70, DEF 35
      - Skeleton King: HP 350, ATK 65, DEF 45

      Floor 10+ (Boss):
      - Mini Boss: HP 500, ATK 80, DEF 50
      - Floor Guardian: HP 800, ATK 100, DEF 60
      - Tower Lord (Final Boss): HP 1500, ATK 120, DEF 80
    </monster_types>

    <key_and_door_system>
      - Yellow key + Yellow door: most common, found throughout the tower
      - Blue key + Blue door: less common, guard better rewards
      - Red key + Red door: rare, guard important areas and powerful items
      - Using a key on a door consumes one key of that color
      - Keys are limited resources - strategic key usage is core to the puzzle
      - Some doors may be one-way (can only pass through from one side)
    </key_and_door_system>

    <item_system>
      <consumables>
        - Small potion: Restore 200 HP (capped at max HP)
        - Big potion: Restore 500 HP (capped at max HP)
        - Super potion: Fully restore HP to max
      </consumables>

      <permanent_upgrades>
        - Red gem: +3 ATK permanently
        - Blue gem: +3 DEF permanently
        - Green gem: +1 ATK and +1 DEF permanently
        - Attack book: +ATK (varies by floor)
        - Defense book: +DEF (varies by floor)
        - Life gem: +Max HP
      </permanent_upgrades>

      <equipment>
        - Iron Sword: +10 ATK
        - Steel Sword: +20 ATK
        - Holy Sword: +50 ATK
        - Wooden Shield: +8 DEF
        - Iron Shield: +15 DEF
        - Holy Shield: +40 DEF
        Equipment replaces previous equipment of same slot (only one sword, one shield)
      </equipment>

      <currency>
        - Gold pile: +50 gold
        - Big gold pile: +200 gold
        - Gold is used in shops to buy items
      </currency>
    </item_system>

    <shop_system>
      - Shop NPCs appear on certain floors
      - Player can buy items with gold:
        - Potions (small: 25 gold, big: 100 gold)
        - Keys (yellow: 10 gold, blue: 50 gold, red: 100 gold)
        - Gems (red: 80 gold, blue: 80 gold)
      - Some shops also buy items from player
      - Shop interface opens as a modal overlay
    </shop_system>

    <npcs_and_events>
      - Guide NPC: gives hints and game tips
      - Healer NPC: fully heals player (limited uses per floor)
      - Teleporter NPC: teleports player to a specific floor
      - Gambling NPC: player pays gold for random item reward
      - Fortune teller: shows monster stats preview
      - Story NPCs: advance the game narrative
      - Trapped floor: stepping on certain tiles triggers damage or teleport
    </npcs_and_events>
  </game_design>

  <key_interactions>
    <movement_flow>
      1. Player presses arrow key / WASD / clicks adjacent tile
      2. If target tile is passable: player moves to tile
      3. If target tile is wall: movement blocked, no action
      4. If target tile has item: item auto-picked up, stats updated, message shown
      5. If target tile has monster: combat initiated automatically
      6. If target tile is door: key consumed, door opens, player moves to tile
      7. If target tile is stairs: floor transition with fade animation
      8. If target tile is NPC: interaction dialogue opens
      9. Each movement counts as one step
    </movement_flow>

    <combat_flow>
      1. Player moves into monster tile
      2. Combat overlay appears showing both combatants
      3. Auto-combat resolves round by round with visual feedback:
         - Damage numbers popup
         - HP bars animate
         - Hit effects on sprites
      4. If player wins: monster disappears, XP/gain shown, player stays on tile
      5. If player loses: game over screen with options
      6. Player can see expected combat result via tooltip before engaging
    </combat_flow>

    <floor_transition_flow>
      1. Player steps on stairs tile
      2. Brief transition animation (fade or slide)
      3. New floor loads with player at corresponding stairs position
      4. Floor number indicator updates
      5. Message log shows "Entered Floor X"
    </floor_transition_flow>

    <shop_flow>
      1. Player interacts with shopkeeper NPC
      2. Shop modal opens showing available items with prices
      3. Player clicks item to buy (if enough gold)
      4. Gold deducted, item added, modal updates
      5. Close shop to resume game
    </shop_flow>
  </key_interactions>

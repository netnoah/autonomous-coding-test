🎯 What Was Implemented

1. ShopModal Component

  • Created src/components/ShopModal.jsx
  • Displays shop items with icons, names, descriptions, and prices
  • Shows player's current gold
  • Buy buttons are disabled when player can't afford items
  • Modal can be closed with X button or "Close Shop" button

2. Shop Items (per specification)

  • Small Potion - 25 gold (+200 HP)
  • Big Potion - 100 gold (+500 HP)
  • Yellow Key - 10 gold (Opens yellow doors)
  • Blue Key - 50 gold (Opens blue doors)
  • Red Key - 100 gold (Opens red doors)
  • Red Gem - 80 gold (+3 ATK)
  • Blue Gem - 80 gold (+3 DEF)

3. Game Integration

  • Added BUY_ITEM action to gameReducer
  • Added CLOSE_SHOP action to gameReducer
  • Shop opens when player moves onto Shopkeeper NPC tile
  • Gold deduction and item addition logic implemented
  • Shop state tracked in game state (shopOpen: false)

🧪 Manual Test Steps

Test 1: Shop Opens When Interacting with Shopkeeper

  1. Open game: http://localhost:3000
  2. Click "New Game"
  3. Navigate to Floor 3 (shop floor):
    • Go up stairs from Floor 0 → Floor 1
  4. Move onto the Shopkeeper NPC (center of Floor 3)
  5. ✓ EXPECTED: Shop modal opens with all items displayed
  6. ✓ EXPECTED: Player's current gold shown (0 initially)
  7. ✓ EXPECTED: All buy buttons disabled (can't afford items)

Test 2: Collect Gold and Buy Items

  1. On Floor 3, collect the 2 gold piles (50 gold each = 100 total)
  2. Move onto Shopkeeper NPC to open shop
  3. Now you should have 100 gold
  4. ✓ EXPECTED: Can afford Yellow Key (10g), Blue Key (50g), Big Potion (100g)
  5. ✓ EXPECTED: Cannot afford Red Key (100g - but leaves you with 0), Red/Blue Gems (80g)
  6. Click "Buy" on Yellow Key
  7. ✓ EXPECTED: Gold decreases to 90, Yellow Keys increases to 2
  8. ✓ EXPECTED: Success message appears: "Purchased Yellow Key!"
  9. Click "Buy" on Big Potion
  10. ✓ EXPECTED: Gold decreases to 0, HP increases by 500

Test 3: Cannot Buy Without Enough Gold

  1. With 0 gold, try to click buy on any item
  2. ✓ EXPECTED: Buy button is disabled (grayed out)
  3. ✓ EXPECTED: Button shows "Not enough gold" text

Test 4: Close Shop

  1. With shop open, click the ✕ button in top-right
  2. ✓ EXPECTED: Shop modal closes
  3. ✓ EXPECTED: Game controls work again
  4. Move onto Shopkeeper again
  5. ✓ EXPECTED: Shop reopens
  6. Click "Close Shop" button at bottom
  7. ✓ EXPECTED: Shop closes again

✅ Code Verification

Files Modified:

Reducer Actions Added:

case 'BUY_ITEM':
  return handleBuyItem(state, action.item)

case 'CLOSE_SHOP':
  return { ...state, shopOpen: false }

Buy Item Logic:

• Checks if player has enough gold
• Deducts gold from player
• Adds item to player's inventory/stats:
  - Potions: Increase HP (capped at max HP)
  - Keys: Increase key count
  - Gems: Increase ATK/DEF
• Shows success message
• Returns error message if not enough gold

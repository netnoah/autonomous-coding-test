✅ COMBAT PREDICTION SHOWS ACCURATE OUTCOME (Test #1595)

Fixed the combat prediction system to provide accurate predictions by using
effective player stats (base + equipment bonuses) instead of just base stats.
Previously, the tooltip showed inaccurate predictions for players with
equipment, leading to unexpected combat results.

**Root Cause Identified:**
- MonsterTooltip was using player.atk and player.def (base stats only)
- Actual combat was using effectiveStats.atk and effectiveStats.def (base + equipment)
- This caused predictions to be inaccurate when player had swords or shields

**Implementation Details:**

1. GameMap Component (src/components/GameMap.jsx):
   - Added `equipment` to destructured gameState
   - Passed equipment prop to MonsterTooltip component

2. MonsterTooltip Component (src/components/MonsterTooltip.jsx):
   - Imported EQUIPMENT_STATS from gameReducer
   - Created getEffectivePlayerStats helper function (matches gameReducer logic)
   - Updated predictCombat to accept and use equipment parameter
   - Calculates effective ATK and DEF including equipment bonuses
   - Displays effective stats in tooltip for transparency
   - Shows equipped items (sword/shield) to player

3. Prediction Formula (verified to match actual combat):
   - playerDmg = Math.max(0, effectiveStats.atk - monster.def)
   - monsterDmg = Math.max(0, monster.atk - effectiveStats.def)
   - roundsToKill = Math.ceil(monster.hp / playerDmg)
   - totalDamageTaken = (roundsToKill - 1) * monsterDmg
   - remainingHP = player.hp - totalDamageTaken

4. UI Enhancements:
   - Added "Your Stats (with equipment)" section
   - Shows effective ATK and DEF values
   - Displays equipped sword and shield names
   - Updated prediction labels for clarity:
     * "Your damage per hit" (was "Your damage")
     * "Enemy damage per hit" (was "Enemy damage")
     * "Rounds to victory" (was "Rounds to kill")
     * "Total damage taken" (was "Damage taken")
     * "HP remaining" (was "Remaining HP")

**Verification:**
- ✅ Prediction logic matches actual combat logic exactly
- ✅ Both use effective stats calculation
- ✅ Damage formulas are identical
- ✅ Round calculation matches combat loop
- ✅ Players now see accurate predictions before fighting

================================================================================
CRITICAL BUG DISCOVERED AND FIXED
================================================================================

❌ ISSUE: Player movement completely non-functional

During verification testing at the start of Session 44, discovered that
both keyboard and mouse click movement were completely broken:
- Arrow keys did not move player
- WASD keys did not move player
- Clicking on adjacent tiles did not move player
- Step counter remained at 0

This was a CRITICAL regression that made the game unplayable.

================================================================================
ROOT CAUSE ANALYSIS
================================================================================

🔍 Problem: handleNPC function only opened dialogue, did not move player

The bug was in the game reducer's handleNPC function. When the player
attempted to move from starting position (5,5) to position (5,4) where
an NPC was located:

1. handleMove detected NPC at target tile
2. Called handleNPC(stateToUse, TILE_TYPES.NPC)
3. handleNPC returned state with dialogue open BUT player position unchanged
4. Player remained at (5,5), steps counter stayed at 0
5. User saw no movement, game appeared frozen

The function signature was:
  function handleNPC(state, npcType)

But it needed to be:
  function handleNPC(state, npcType, newX, newY, direction)

And it needed to update player position in returned state.

================================================================================
FIX IMPLEMENTED
================================================================================

✅ Modified Files:

1. src/components/Game.jsx:
   - Added 'dispatch' to keyboard useEffect dependencies
   - Ensures event handlers have access to latest dispatch function

2. src/game/gameReducer.jsx:
   - Updated handleNPC function signature to accept newX, newY, direction
   - Modified NPC interaction to move player to NPC position
   - Modified shopkeeper interaction to move player to shopkeeper position
   - Player steps counter now increments correctly when moving to NPCs

**Code Changes:**

```javascript
// Before:
function handleNPC(state, npcType) {
  // Only opened dialogue, did not move player
  return { ...state, npcDialogueOpen: true, ... }
}

// After:
function handleNPC(state, npcType, newX, newY, direction) {
  return {
    ...state,
    player: {
      ...state.player,
      x: newX,
      y: newY,
      direction: direction,
      steps: state.player.steps + 1
    },
    npcDialogueOpen: true,
    ...
  }
}
```
